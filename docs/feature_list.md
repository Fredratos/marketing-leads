# 小红书海外营销线索平台 — Feature List

## 模块1：数据采集引擎

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| DC-01 | 关键词管理 | 管理员可创建/编辑/启用/停用监控关键词组合（如"海外达人营销""海外品牌出海""KOL 海外"等），支持词组与排除词 | 🔴 P0 | |
| DC-02 | 平台对接采集 | 对接小红书公开内容接口/搜索，按关键词定时拉取匹配帖子列表 | 🔴 P0 | 需处理反爬/频率限制 |
| DC-03 | 帖子详情采集 | 对匹配帖子，采集完整正文、图片OCR文字、发布人信息、发布时间、互动数据（点赞/收藏/评论数） | 🔴 P0 | |
| DC-04 | 评论采集 | 采集目标帖子的全部评论内容、评论人、评论时间 | 🔴 P0 | |
| DC-05 | 智能线索识别 | 基于 NLP 规则/LLM 自动判断帖子是否为有效线索（有人找资源/有人提问/有人分享经验），排除纯广告/无关内容 | 🔴 P0 | |
| DC-06 | 采集任务调度 | 支持设置采集频率（每小时/每天/每周），查看采集任务日志与状态 | 🟡 P1 | |
| DC-07 | 重复去重 | 同一帖子多次采集时自动去重，避免重复保存 | 🟡 P1 | |

## 模块2：线索管理看板

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| LM-01 | 线索列表 | 以列表形式展示所有采集到的线索，包含：帖子标题、发布人、线索类型、采集时间、跟进状态 | 🔴 P0 | |
| LM-02 | 线索筛选搜索 | 按关键词、发布时间、线索类型、跟进状态等维度**组合筛选**（AND逻辑）；支持全文搜索 | 🔴 P0 | 筛选条件支持多选组合 |
| LM-03 | 线索详情页 | 展示完整帖子内容（含图片）、发布人信息、所有评论内容、互动数据、系统判定线索类型的依据 | 🔴 P0 | |
| LM-04 | 线索状态管理 | 支持手动标记线索状态：新线索/已查看/已联系/已转化/无效；支持批量操作 | 🔴 P0 | |
| LM-05 | 跟进记录 | 对每条线索添加跟进备注、下次联系时间 | 🟡 P1 | |
| LM-06 | 线索导出 | 支持选中线索导出为 Excel/CSV | 🟡 P1 | |
| LM-07 | 收藏/标记 | 对重点线索加星标或颜色标记 | 🟢 P2 | |

## 模块3：统计分析

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| ST-01 | 概览仪表盘 | 展示关键指标：线索总量、各状态数量、趋势图、热门关键词词云 | 🟡 P1 | |
| ST-02 | 采集效果分析 | 展示各关键词组合的命中率、有效线索率 | 🟢 P2 | |
| ST-03 | 团队协作统计 | 多用户场景下，统计各成员的线索跟进情况 | 🟢 P2 | |

|---------|---------|---------|--------|------|

## 优先级定义

| 优先级 | 含义 | 标识 |
|-------|------|------|
| P0 | MVP 必须，核心功能 | 🔴 |
| P1 | 重要增强，首版应有 | 🟡 |
| P2 | 体验优化，可延后 | 🟢 |

- [x] 线索管理 → 有状态流转和跟进记录
